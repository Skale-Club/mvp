<objective>
## Goal
Every outbound notification (Twilio SMS, Resend email, GoHighLevel contact sync) is persisted to `notification_logs` with recipient, preview, status, and — when available — `leadId` and `providerMessageId`. Logging is non-blocking: a failure to log NEVER breaks the original send.

## Purpose
Plan 01-01 created the data layer. This plan wires it up at the point where notifications actually leave the system. After this plan, every SMS, email, and GHL sync leaves a trail. API endpoints and admin UI (plans 01-03 / 01-04) will read from this trail.

## Output
- New file: `server/notifications/logger.ts` exporting a safe `logNotification()` helper
- Modified integration functions (`sendHotLeadNotification`, `sendAbandonedLeadNotification`, `sendNewChatNotification`, `sendLowPerformanceAlert`, `sendNewLeadNotification`, `sendContactFormNotification`, `getOrCreateGHLContact`) each emitting one log row per recipient / per attempt
- Modified call sites (`server/routes/leads.ts`, `server/routes/chat.ts`, `server/leads/abandonedNotifications.ts`) passing `{ leadId, trigger }` context
- Typecheck clean (`npm run check`)
</objective>

    Create a new file `server/notifications/logger.ts` exporting a single function:

    ```ts
    import type { InsertNotificationLog } from "#shared/schema.js";
    import { storage } from "../storage.js";

    const MAX_PREVIEW_CHARS = 5000;

    export async function logNotification(entry: InsertNotificationLog): Promise<void> {
      try {
        const safeEntry: InsertNotificationLog = {
          ...entry,
          preview: typeof entry.preview === "string" && entry.preview.length > MAX_PREVIEW_CHARS
            ? entry.preview.slice(0, MAX_PREVIEW_CHARS)
            : entry.preview,
        };
        await storage.createNotificationLog(safeEntry);
      } catch (error) {
        console.error("Failed to write notification log:", error);
      }
    }

    export type NotificationLogContext = {
      leadId?: number | null;
      trigger: string;
      recipientName?: string | null;
      metadata?: Record<string, unknown> | null;
    };
    ```

    Avoid:
    - Do NOT re-throw on insert failure. The log is audit-only; a failing log must never take down the user-facing flow.
    - Do NOT add a default `trigger` inside the helper. Callers must pass it explicitly via context (enforced by TS on `InsertNotificationLog`).
    - Do NOT introduce retry logic here — deferred to a future milestone.
  </action>
  <verify>`npm run check` passes; the file exports `logNotification` and `NotificationLogContext`</verify>

    2. Modify the internal `sendSms(config, body, logParams)` helper to accept a new third parameter:
       ```ts
       async function sendSms(
         config: TwilioConfig,
         body: string,
         logParams: { trigger: string; leadId?: number | null; metadata?: Record<string, unknown> | null },
       ): Promise<TwilioResult> {
         try {
           const twilio = await import("twilio");
           const client = twilio.default(config.accountSid, config.authToken);
           for (const to of config.recipients) {
             try {
               const result = await client.messages.create({ body, from: config.from, to });
               await logNotification({
                 channel: "sms",
                 trigger: logParams.trigger,
                 recipient: to,
                 preview: body,
                 status: "sent",
                 providerMessageId: result?.sid || null,
                 leadId: logParams.leadId ?? null,
                 metadata: logParams.metadata ?? null,
               });
             } catch (perRecipientError: any) {
               await logNotification({
                 channel: "sms",
                 trigger: logParams.trigger,
                 recipient: to,
                 preview: body,
                 status: "failed",
                 errorMessage: perRecipientError?.message || "Unknown error",
                 leadId: logParams.leadId ?? null,
                 metadata: logParams.metadata ?? null,
               });
               throw perRecipientError;
             }
           }
           return { success: true };
         } catch (error: any) {
           console.error("Failed to send Twilio SMS:", error);
           return { success: false, message: error?.message || "Failed to send SMS" };
         }
       }
       ```
       Key difference: move try/catch to a per-recipient scope so partial failures are recorded. Rethrow so outer catch still returns the same `{ success: false, message }` shape callers expect (AC-5).
