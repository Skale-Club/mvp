<objective>
## Goal
Show at-a-glance notification delivery status for each lead directly on the leads list — a new compact column with per-channel icons (email, SMS) colored by the most recent send status, plus a popover revealing recipient / time / error / preview snippet on click.

## Purpose
Operator opens /admin/leads and immediately sees whether notifications are firing for every lead. No new admin section, no modal panels — the indicator lives in the row. Chat and low-performance notifications are not displayed (operator doesn't care about those in the leads workflow).

- `client/src/components/admin/notification-indicators.tsx` — already extracted during course-correction: keep as the shared home for `ChannelBadge`, `StatusBadge`, `NotificationPreview`

- Typecheck clean; browser verified
</objective>

    3. Inside the `LeadsSection` component, add a single notification-log query near the existing `useQuery<FormLead[]>` call:
       ```ts
       const { data: allLogs = [] } = useQuery<NotificationLog[]>({
         queryKey: ['/api/admin/notification-logs', { limit: 500 }],
         queryFn: async () => {
           const res = await apiRequest('GET', '/api/admin/notification-logs?limit=500');
           return res.json();
         },
         staleTime: 30_000, // cache for 30s to avoid re-fetch on tab switches
       });
       ```

    4. Build a memoized summary map:
       ```ts
       const notificationsByLead = useMemo(() => {
         const map = new Map<number, { email?: NotificationLog; sms?: NotificationLog; ghl?: NotificationLog }>();
         for (const log of allLogs) {
           if (log.leadId == null) continue;
           const existing = map.get(log.leadId) ?? {};
           const slot: 'email' | 'sms' | 'ghl' | null =
             log.channel === 'email' ? 'email'
             : log.channel === 'sms' ? 'sms'
             : log.channel === 'ghl_sync' ? 'ghl'
             : null;
           if (!slot) continue;
           const current = existing[slot];
           if (!current || new Date(log.sentAt as any) > new Date(current.sentAt as any)) {
             existing[slot] = log;
           }
           map.set(log.leadId, existing);
         }
         return map;
       }, [allLogs]);
       ```

    5. In the leads TABLE render block (find it via grep `<Table>` or `<thead>` or the existing column headers; it's the main leads list, NOT the modal), add a new column:

       - In each row, render `<NotificationIconCell summary={notificationsByLead.get(lead.id) ?? {}} />`
