# Plan: Limitar upload em lote da galeria (concorrência de 5)

## Problema
Em `client/src/components/admin/GallerySection.tsx:423`, a função `uploadFiles` usa `Promise.all` para disparar **todas** as uploads simultaneamente. Com 120 fotos, isso sobrecarrega o servidor e o navegador.

## Solução
Substituir o `Promise.all` por uma função de processamento em lotes que executa no máximo **5 uploads simultâneas**.

## Alterações

### 1. `client/src/components/admin/GallerySection.tsx`

Adicionar uma função utilitária `processInBatches` (ou usar uma abordagem inline com concorrência controlada) antes ou dentro do componente.

**Linha ~423**: Substituir o `Promise.all` por uma lógica de fila com concorrência máxima de 5:

```ts
// Helper: processa itens com concorrência máxima
async function processInBatches<T, R>(
  items: T[],
  concurrency: number,
  fn: (item: T, index: number) => Promise<R>
): Promise<R[]> {
  const results: R[] = new Array(items.length) as R[];
  let nextIndex = 0;

  async function worker() {
    while (nextIndex < items.length) {
      const currentIndex = nextIndex++;
      results[currentIndex] = await fn(items[currentIndex], currentIndex);
    }
  }

  const workers = Array.from(
    { length: Math.min(concurrency, items.length) },
    () => worker()
  );
  await Promise.all(workers);
  return results;
}
```

Depois, na linha 423, trocar:
```ts
const outcomes = await Promise.all(
  pendingItems.map(async (item, index) => { ... })
);
```

Por:
```ts
const outcomes = await processInBatches(pendingItems, 5, async (item, index) => {
  // ... mesmo corpo do callback atual
});
```
