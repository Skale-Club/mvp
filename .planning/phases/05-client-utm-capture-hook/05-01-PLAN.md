export function useAttribution(): { visitorId: string | null } {
  const [visitorId, setVisitorId] = useState<string | null>(null);

  // Effect A â€” mount-only: capture UTMs / classify referrer / first session ping

Two-useEffect implementation (per D-03 â€” this is non-negotiable):

**Effect A â€” `useEffect(() => { ... }, [])`** (runs ONCE on mount, before any Wouter navigation can destroy `window.location.search`):

1. Resolve visitor ID: `const id = ensureVisitorId();` then `setVisitorId(id);`
2. Read URL params synchronously: `const params = new URLSearchParams(window.location.search);`
3. Read each UTM with normalize: define a tiny helper `const norm = (v: string | null) => { if (!v) return undefined; const t = v.toLowerCase().trim(); return t.length > 0 ? t : undefined; };` Apply to: `utm_source`, `utm_medium`, `utm_campaign`, `utm_term`, `utm_content`, `utm_id`.
4. Capture `const referrer = document.referrer || '';`
5. Determine first-touch attribution:
   - If `utm_source` is present (truthy after normalize): use UTM values directly. Set `ftSource = utm_source`, `ftMedium = utm_medium ?? '(not set)'`, `ftCampaign = utm_campaign`, `ftTerm = utm_term`, `ftContent = utm_content`, `ftId = utm_id`, `ftSourceChannel = channelFromUtmMedium(utm_medium)`.
   - If NO `utm_source`: call `const cls = classifyReferrer(referrer);` and use `ftSource = cls.ftSource`, `ftMedium = cls.ftMedium`, `ftSourceChannel = cls.sourceChannel`.
6. Build the session-ping payload (matches `InsertVisitorSession` shape minus id/timestamps):
   ```typescript
   const payload = {
     visitorId: id,
     ftSource, ftMedium, ftCampaign, ftTerm, ftContent, ftId,
     ftSourceChannel,
     ftLandingPage: window.location.pathname,
     ftReferrer: referrer || undefined,
     // lt_* mirrors ft_* on first ping â€” server upsert preserves ft_* on subsequent calls
     ltSource: ftSource,
     ltMedium: ftMedium,
     ltCampaign: ftCampaign,
     ltTerm: ftTerm,
     ltContent: ftContent,
     ltId: ftId,
     ltSourceChannel: ftSourceChannel,
     ltLandingPage: window.location.pathname,
     ltReferrer: referrer || undefined,
     deviceType: detectDeviceType(),
   };
   ```
   Strip undefined keys before posting (server schema rejects empty strings as Zod min-length, undefined is allowed since fields are optional). Use a small reducer or `Object.fromEntries(Object.entries(payload).filter(([_, v]) => v !== undefined))`.
7. `postSessionPing(payload);` â€” fire-and-forget, never await.
8. Fire the FIRST `reportAttributionPageView(window.location.pathname, id);` here too â€” Effect B will skip the initial render because `visitorId` is null on the very first run before `setVisitorId` flushes; firing here guarantees the landing page is recorded as a page_view.

**Effect B â€” `useEffect(() => { ... }, [location])`** (runs on every Wouter route change):

1. Guard: `if (!visitorId) return;` â€” Effect A handles the first render's page_view; this effect only handles subsequent navigations.
2. Determine last-touch from CURRENT URL: re-read `new URLSearchParams(window.location.search)` (Wouter does support query strings on some routes, but typically ad clicks land on root). If `utm_source` is present, override lt_*; otherwise leave lt_* untouched (do NOT overwrite with classified referrer here â€” the referrer at navigation time is the previous in-app page, which is meaningless for last-touch).
3. Build minimal payload:
   ```typescript
   const params = new URLSearchParams(window.location.search);
   const utmSource = norm(params.get('utm_source'));
   const ltPayload: Record<string, unknown> = {
     visitorId,
     ltLandingPage: location,
   };
   if (utmSource) {
     ltPayload.ltSource = utmSource;
     ltPayload.ltMedium = norm(params.get('utm_medium'));
     ltPayload.ltCampaign = norm(params.get('utm_campaign'));
     ltPayload.ltTerm = norm(params.get('utm_term'));
     ltPayload.ltContent = norm(params.get('utm_content'));
     ltPayload.ltId = norm(params.get('utm_id'));
     ltPayload.ltSourceChannel = channelFromUtmMedium(params.get('utm_medium'));
   }
   // Strip undefined values, then:
   postSessionPing(ltPayload);
   reportAttributionPageView(location, visitorId);
   ```
   The `norm` helper is the same as Effect A â€” define it once at the top of the hook function or inline at module scope (export-less helper).

Critical rules (from CONTEXT.md):
- Effect A's dependency array is EXACTLY `[]` (per D-03). Do NOT include `visitorId`, `location`, or any other dependency. ESLint react-hooks/exhaustive-deps will warn but it must be SUPPRESSED with `// eslint-disable-next-line react-hooks/exhaustive-deps` comment IF the project has the rule enabled â€” check existing hooks (use-auth.ts is too small to confirm; `git grep eslint-disable client/src/hooks/`).
