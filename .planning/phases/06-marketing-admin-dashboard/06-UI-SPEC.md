## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gaps, badge inner padding |
| sm | 8px | Filter chip gaps, table cell padding (py-2), compact button padding |
| md | 16px | Card inner padding (`p-4`), filter bar gap between controls |
| lg | 24px | Section header bottom border clearance (`pb-5` = 20px, nearest is `pb-6` = 24px — use `pb-5` to match `AdminPageHeader`) |
| xl | 32px | Gap between filter bar and tab strip (`mt-8`) |
| section | 24px | `space-y-6` between major layout rows (matching `DashboardSection.tsx` pattern) |
| card-pad | 20px | `p-5` on KPI cards and chart cards (matches existing `DashboardSection.tsx` pattern exactly) |

Exceptions:
- `AdminPageHeader` uses `pb-5` (20px) — inherit as-is to match all other sections.
- KPI cards use `p-5` (20px) — matches existing admin dashboard convention; do not normalize to 24px.
- Touch targets: filter preset buttons minimum height `h-9` (36px); `<Select>` triggers use `h-9` — matches `LeadsSection.tsx` filter row pattern.

## Typography

Source: `client/src/index.css` CSS variables + observed admin component conventions.

Exactly 2 font weights are used across this phase: 400 (normal) and 700 (bold).

| Role | Size | Weight | Line Height | Font | Usage |
|------|------|--------|-------------|------|-------|
| Body | 14px (`text-sm`) | 400 (normal) | 1.5 | Inter | Table cell text, filter labels, card body text |
| Label / Caption | 12px (`text-xs`) | 400 (normal) | 1.4 | Inter | KPI card label (uppercase + tracking-wide), table column headers, badge text |
| Heading (card) | 16px (`text-base`) | 700 (bold) | 1.3 | Inter | Card section headings (`text-base font-bold`) |
| Heading (page) | 24px (`text-2xl`) | 700 (bold) | 1.2 | Outfit | `AdminPageHeader` title (`text-2xl font-bold`) |
| Display / KPI value | 24px (`text-2xl`) | 700 (bold) | 1.1 | Inter | KPI metric numbers |

All font sizes are Tailwind utility classes. Do not hardcode `px` values in JSX — use utility classes only.

## Color

Source: `client/src/index.css` CSS variables + `CLAUDE.md` brand guidelines + `06-CONTEXT.md` decisions D-20, D-27.

| Role | Value | Tailwind / CSS var | Usage |
|------|-------|-------------------|-------|
| Dominant (60%) | `hsl(0 0% 100%)` white | `bg-background` | Page background, card backgrounds |
| Secondary (30%) | `hsl(210 40% 96.1%)` slate-50 | `bg-muted` / `bg-card` / `--section` | Admin section background, tab list background, table row hover |
| Primary accent (10%) | `#1C53A3` → `hsl(215 71% 37%)` | `bg-primary` / `text-primary` | Active tab indicator, active date preset button (variant="default"), chart "Visits" area series, primary CTA buttons |
| Brand yellow | `#FFFF01` → `hsl(60 100% 50%)` | `bg-secondary` | Chart "Conversions" area series fill. Use `#FFD700` (gold) for chart stroke on white backgrounds for legibility — note this in chart implementation. DO NOT use for interactive controls (fails contrast). |
| Destructive | `hsl(0 84.2% 60.2%)` | `bg-destructive` / `text-destructive` | Not used in Phase 6 (no destructive actions). Reserved for future error states only. |
| Muted text | `hsl(215.4 16.3% 46.9%)` | `text-muted-foreground` | Labels, captions, empty state body, helper text |
| Success (HOT badge) | `hsl(142 76% 36%)` | `bg-green-100 text-green-800` | HOT lead quality badge (D-20) |
| Warning (WARM badge) | `hsl(38 92% 50%)` | `bg-amber-100 text-amber-800` | WARM lead quality badge (D-20) |
| Muted (COLD badge) | slate gray | `bg-gray-100 text-gray-700` | COLD lead quality badge (D-20) |

Accent (`bg-primary`) reserved for:
1. Active date-preset button (`variant="default"`)
2. Recharts AreaChart "Visits" series fill + stroke
3. Sidebar active nav item (existing admin pattern — do not change)
4. KPI card icon container accent highlight (optional — follow DashboardSection.tsx: `text-blue-500`)

Never apply primary blue to: table row backgrounds, badge backgrounds, or static text.

Conversion event type pill colors (D-27):
- Lead Created: `bg-green-100 text-green-800`
- Phone Call: `bg-blue-100 text-blue-800`
- Form Submitted: `bg-purple-100 text-purple-800`
- Booking Started: `bg-orange-100 text-orange-800`

## Filter Bar Interaction Contract

Source: D-06, D-07, D-08, D-09, D-10.

| Behavior | Specification |
|----------|--------------|
| Default filter state | `{ datePreset: '30d' }` — matches server 30-day default |
| Active date preset visual | `variant="default"` (primary blue background, white text) |
| Inactive date preset visual | `variant="outline"` (border, muted text) |
| Date preset button height | `h-9` (36px) via `size="sm"` Button prop |
| Select trigger width | Source: `w-[180px]`; Campaign: `w-[200px]`; Conversion Type: `w-[200px]` |
| Select placeholder | "All sources" / "All campaigns" / "All types" |
| Filter change behavior | Instant — updates `queryKey`, triggers React Query refetch. No "Apply" button. |
| Custom date range | Opens `<Popover>` with `react-day-picker` range calendar. Trigger shows formatted range: "Apr 1 – Apr 25" or "Select dates" when unset. |
| Custom preset active state | Popover trigger button uses `variant="default"` only when both `dateFrom` and `dateTo` are set. |
| Source dropdown population | Populated from unique sources in current API response. Regenerated when date range changes. |
| Campaign dropdown population | Populated from unique campaigns in current API response. Same behavior. |
| Filter bar placement | Below `AdminPageHeader`, above tab strip. Not collapsible. Always rendered. |

## Interaction States

| State | Behavior |
|-------|----------|
| Loading (first fetch) | Skeleton layout (see Loading State Specification above) |
| Refetching (filter change) | Table/chart remains visible; React Query background refetch. No spinner overlay unless > 2s (add `isFetching` opacity: `opacity-60 pointer-events-none` on tab content during background fetch if desired). |
| Empty (zero rows returned) | Coach-mark empty state (see Empty State Specification) |
| API error | Replace tab content with error card: heading + retry button that calls `refetch()` |
| Active filter preset | `variant="default"` button |
| Table row | `hover:bg-muted/40 transition-colors` |
| Chart tooltip | Recharts default `<Tooltip>` with custom `contentStyle`: `{ background: 'hsl(var(--popover))', border: '1px solid hsl(var(--border))', borderRadius: '6px', fontSize: '13px' }` |
