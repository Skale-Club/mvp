<objective>
Append a "Marketing Attribution" panel to the existing Lead detail Dialog in `LeadsSection.tsx` so a business owner who opens any lead can immediately see how that lead found the business — in plain language, no `utm_` field names.

The panel renders six pieces of attribution data using the same `DetailItem` component already used throughout the dialog:
- First Source (channelLabel of `selectedLead.firstTouchSource`)
- First Campaign (`selectedLead.firstTouchCampaign` or "—")
- Last Source (channelLabel of `selectedLead.lastTouchSource`)
- Last Campaign (`selectedLead.lastTouchCampaign` or "—")
- Landing Page (`selectedLead.ftLandingPage` from Plan 07-01 server enrichment, or "—")
- Visits Before Conversion (`selectedLead.visitCount` from Plan 07-01, or "—" if 0/missing)

Per D-14, the entire panel is omitted for leads with `firstTouchSource === null` (pre-attribution leads from before v1.2). No empty panel, no "—" wasteland.

Purpose: Closes LEADATTR-01 and LEADATTR-02. With Plan 07-02 covering DASH-06, this plan completes the phase.

Output: A single file modified — `client/src/components/admin/LeadsSection.tsx`. No new files, no schema changes, no new server routes.
</objective>
